## Context

The discovery + the choice of which learnings to keep happen in `/reflect` (the
Agent-Learning-Loop). This command is the SAFE PERSIST half: it lands one (or a few)
already-approved learnings on `main` without breaking governance or bloating a dense file.

## Your task

Persist the approved learning(s) at the narrowest fitting level, with the diff previewed before
any write and the merge gated on real CI.

### Steps

1. **Pick the narrowest level.** A = this project's `CLAUDE.md`; B = the relevant global
   `~/.claude/agents/<agent>.md` (then re-run `./build-explorer.sh`); C = a new skill
   (`~/.claude/skills/<name>/SKILL.md`). Prefer A > B > C.
2. **De-duplicate / consolidate, don't bloat.** If the learning sharpens or repeats an existing
   rule, CONSOLIDATE (merge/prune to one first-class statement + incident pointers) rather than
   adding a near-duplicate — a dense file loses signal when padded.
3. **Preview the exact diff FIRST.** Show the before/after of the rule + its target level; write
   ONLY on an explicit human "y". Never write shared/global config blindly.
4. **Branch from main** (`docs/<slug>` or the active feature branch — never commit straight to a
   shared branch).
5. **Write + verify.** Apply the edit; `bash config/claude/tests/run_all.sh` → ALL CHECKS PASSED.
   (A new `docs/**.md` starts with a plain `#` and NO `---` frontmatter, or the frontmatter
   validator demands name/description; don't introduce an unquoted `mcp__<family>__` literal
   whose family isn't in DEPENDENCIES.md.)
6. **Commit + PR + watch CI.** Commit with a recognized type (`fix:`/`feat:` if it should appear
   in the changelog; `docs:`/`chore:` to keep it out). Push, open the PR, and **watch CI by the
   pushed HEAD sha** (`gh run list --json headSha` matched to `git rev-parse HEAD`, or
   `gh pr checks <PR> --watch`) — never the first/most-recent run, which is often the prior push's.
7. **Merge + sync.** Confirm `ci conclusion=success` on EVERY OS (macOS is strict; a
   release-please-style branch gets no CI → run `run_all.sh` on it locally first). Merge NORMAL
   (no squash unless necessary). `git checkout main && git pull`; confirm `run_all.sh` green on
   `main` after the merge.

### Guardrails

- **Never write shared/global config without previewing the diff + an explicit human y.**
- **Consolidate over add** — sharpen/merge existing rules; a near-duplicate dilutes the file.
- **Merge on `conclusion=success`, not `mergeable`, on every OS — never on RED.**
- **Truth through awareness** — if you skip or shortcut a step, NAME it as an explicit, conscious
  decision rather than passing it silently; an unexamined skip quietly turns implicit habit into
  an ungoverned rule.
