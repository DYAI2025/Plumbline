## Context

Plumbline crosses the OpenRouter real boundary only through the vetted primitives
(`config/claude/lib/council_inference.py`, `config/claude/lib/deepseek_review.py`, the GUI
proxy). This runs the headline real-boundary smoke the way the project requires: real, honest,
key-safe, credit-careful — and recorded at its TRUE evidence class. It is the opposite of a
demo/fake: it crosses the boundary or it honestly reports that it could not.

## Your task

Run ONE live smoke for the target boundary, capture the REAL result verbatim, and record it
honestly. Never re-run until the result looks good.

### Steps

1. **Probe reachability FIRST — immediately before the run.** Free-tier reachability shifts
   minute-to-minute. `GET https://openrouter.ai/api/v1/models` (public, no key) to confirm the
   target model ids are in the live catalog, and/or send a tiny per-model probe; pick the
   baseline/council from *currently-reachable* models. Never assume a model is up; never hardcode
   an id from memory.
2. **Load the key safely.** `OPENROUTER_API_KEY` from `~/.openclaw/.env`, exported only into the
   run's env, used ONLY in the `Authorization: Bearer` header by the vetted transport. Never
   print/echo/log it; never pass it into a subagent.
3. **Gate the live call.** Arm with `COUNCIL_INFERENCE_LIVE=1` + the entrypoint's `--live`
   (+ a `--max-calls` ceiling for council runs). Confirm offline/default fires 0 calls.
4. **Run ONE pass, no re-roll.** Capture the per-role / per-call result verbatim (positions,
   classified codes, usage where exposed). Do NOT re-run until survivors appear or the delta
   looks favorable — that is p-hacking.
5. **Leak-check.** Grep the captured response AND any server log for a key sentinel → must be 0.
6. **Record honestly.** Write `docs/benchmarks/<date>-<feature>-smoke.md` with the real numbers
   and honest attrition. Update `docs/reality/<feature>.evidence.jsonl`: `real-boundary-smoke`
   ONLY for what actually crossed the boundary; everything else stays `integration-fake`. Frame
   it as a wiring/capability smoke, NOT a quality/value verdict. Then verify git is clean +
   `run_all.sh` is green (bench isolation).

### Guardrails (hard-won, binding)

- **Key never leaves the server / never leaked** — header-only, from `~/.openclaw/.env`, never in
  output / log / trace / a subagent's env.
- **No fake fallback.** If a model 429s or is unavailable, surface the classified code
  (`COUNCIL_RATE_LIMITED` / `COUNCIL_MODEL_UNAVAILABLE`) — never a demo/canned result.
- **No credit-spend beyond what is agreed** — name the budget / `--max-calls` before any paid
  run; default to free models; don't probe completions "just to check" on paid ids.
- **Reachable ≠ invocable** — a catalog hit proves reachability; a completion proves invocability.
  Classify precisely; don't let one stand for the other.
- **One run, no re-roll** — report the honest outcome, including high attrition / "underpowered" /
  0 survivors. The pilot's value can be the cost/flakiness finding, not a verdict.
- **Honest ledger class** — `real-boundary-smoke` only when the real boundary was crossed; never
  raise a class to clear a floor.
- **Disclose intermittency** — stamp the reachability context + that a re-run may differ
  (free-tier rate limits, the macOS-CI loopback caveat) in the write-up.
