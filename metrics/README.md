# metrics/

The data layer for the `/agileteam` meta-meta governance
(see `docs/agileteam-governance.md`).

## Files

- **`runs.jsonl`** — append-only, one JSON record per `/agileteam` run. Written by
  `config/claude/metrics/emit_run.py` at the end of Phase 3. This is the spine of drift
  attribution: each record carries the metrics AND a `config_fingerprint` (per-component
  content hashes of the command, agents, and the konfabulations-audit skill). *Not*
  created until the first run is emitted.
- **`process-health.md`** — generated by `config/claude/metrics/process_health.py`:
  SPC per metric + component attribution + a human-gated counter-steering proposal.
- **`bench-<date>.md`** — generated by the `/agileteam-bench` command: frozen `main` vs.
  `agileteam-improved`, arm means + deltas vs. noise threshold.

## Quick start

```bash
# 1) emit a run record (normally the orchestrator does this in Phase 3)
python3 config/claude/metrics/emit_run.py \
  --corpus-id bench-core-v1 --mode core \
  --metrics '{"first_pass":0.72,"mutation":0.81,"unverified_claims":0.04,"cycle_time":3.1}' \
  --gate-outcomes '{"A":"pass","C":"pass"}' --human-overrides 0

# 2) analyse process health (needs a few runs to be meaningful)
python3 config/claude/metrics/process_health.py
```

## Branch note

Per the branch strategy, `runs.jsonl` accumulates on the **`agileteam-improved`** branch
(the evolving process). `main` stays the frozen baseline. For a valid bench comparison,
pin the agent snapshot across both arms — the "frozen main" is only frozen w.r.t.
process/skill files; agent files still evolve, so the baseline must be a pinned
commit/tag, not the moving `main` tip.
