## Context
Plumbline's identity is *measure, don't assert*: every claim earns a deterministic bench.
Bench runs are long, and the manual loop "wait for the slice to finish → show the
per-arm-model summary" was typed by hand ~20 times. This skill automates that watch-and-report
loop. Tooling lives in `config/claude/metrics/` (`emit_run.py`, `process_health.py`,
`challenge_token_oracle.py`) and `metrics/`; results accrue in `metrics/runs.jsonl` on the
`agileteam-improved` branch — `main` stays the frozen baseline.

## Your Task
Kick off (or attach to) a benchmark slice, poll until it finishes, then print the standard
two-metric summary. Never headline catch-rate without the cry-wolf number beside it.

### Steps
1. **Scope the run.** Ask (if not given) which corpus/slice + mode (`core|full`), and confirm
   the **agent snapshot is pinned to a commit/tag across BOTH arms** — agent files keep
   evolving on `main`, so an unpinned comparison is invalid.
2. **Isolate.** Stage all bench inputs OUTSIDE the repo (`/tmp/…` or a dedicated worktree),
   and hard-constrain builder/coder sub-agents to TEXT-ONLY output (no Write/Edit/Bash).
3. **Launch in the background; poll, don't block.** Pick a poll interval matched to how fast
   the run state actually changes; when the harness can notify on completion, prefer that over
   tight polling.
4. **Report on completion.** From `metrics/runs.jsonl` / `process_health.py`, emit per
   **arm × model**: BOTH catch-rate **and** false-positive (cry-wolf) rate, with `n=`, task
   count, and model scope visible. Surface any RED verbatim.
5. **Settle.** Verify `git status` is clean and `bash config/claude/tests/run_all.sh` is green;
   revert any stray files the run wrote into the tree.

### Guardrails
- **Never headline catch-rate alone** — both anti-Goodhart metrics, or the claim misleads.
  "Strictly better" requires holding catch AND not raising cry-wolf — gated on BOTH.
- **Bench isolation:** builder agents TEXT-ONLY, inputs staged outside the repo, clean tree +
  green `run_all` after (in-tree builder agents have reddened `run_all` twice before).
- **Never downgrade a RED or fabricate a pass;** mark absent tooling `MISSING`.
- Pin the agent snapshot across arms; `runs.jsonl` accrues on `agileteam-improved`, not `main`.
