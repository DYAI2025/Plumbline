# Ausgabe-Templates fuer Der Systemdenker

## 1. Standard

```markdown
# Der Systemdenker

## Objekt
[Was wird analysiert?]

## Systembefund
[zu lokal gedacht / teilweise gekoppelt / systemisch durchdacht]

## Systemkontext
- [Betroffene Teile: Nutzer / Daten / Technik / Organisation / Markt]

## Kopplungen / versteckte Abhaengigkeiten
- [Abhaengigkeit zwischen Systemteilen]

## Wechselwirkungen / Feedbackschleifen
- [Rueckwirkung / verstaerkende oder daempfende Schleife]

## Fehlanreize / Drift / Kippunkte
- [Anreiz / Skalierungsbruch / Langzeitfolge]

## Mein blinder Fleck (Complexity Bias)
- [Wo meine Analyse zu komplex wird oder die Umsetzung blockiert]

## Drei wichtigste Systemhebel
- [Hebel 1 + Wirkung]
- [Hebel 2 + Wirkung]
- [Hebel 3 + Wirkung + Hand-off]
```

## 2. Kurz

```markdown
# Der Systemdenker

**Systembefund:** [direkt]
**Staerkste Kopplung:** [eine]
**Kritische Rueckwirkung:** [eine]
**Mein blinder Fleck:** [Complexity Bias konkret]
**Drei Systemhebel:** [die drei wichtigsten in einer Zeile]
```

## 3. Multi-Agent-Diskussion

```markdown
# Der Systemdenker

## Wo zu lokal gedacht wird
[Wo entscheidet die Runde gerade isoliert?]

## Aufgedeckte Systemwirkung
- [Kopplung / Rueckwirkung / Kippunkt]

## Mein blinder Fleck (Complexity Bias)
- [konkret]

## Anschlussbedingung (drei Hebel)
- [unter welcher reduzierenden Bedingung die Analyse in die Umsetzung darf]
```
