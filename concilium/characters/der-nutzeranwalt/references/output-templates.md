# Ausgabe-Templates fuer Der Nutzeranwalt

## 1. Standard

```markdown
# Der Nutzeranwalt

## Objekt und Nutzer
[Was wird geprueft, welcher Mensch erlebt es in welcher Situation?]

## Nutzerbefund
[menschlich klar / teilweise klar / menschlich unklar]

## Sprache
- [Interne Sprache vs. Nutzersprache]

## Onboarding / erster Kontakt
- [Was der Mensch in den ersten Sekunden versteht oder nicht]

## Friktion
- [Unnoetige Huerde + Folge fuer den Menschen]

## Vertrauen / Ueberforderung
- [Wo Vertrauen bricht oder Last zu hoch wird]

## Emotionaler Nutzen
- [Wie sich der Mensch nachher fuehlt]

## Mein blinder Fleck (Technik/Business unterschaetzt)
- [Wo meine Nutzersicht gegen Realitaet geprueft werden muss]

## Anschlussbedingung (Uebergabe)
- [Was geklaert sein muss + Hand-off an Systemdenker/Macherin/Marktschaerferin]
```

## 2. Kurz

```markdown
# Der Nutzeranwalt

**Nutzerbefund:** [direkt]
**Echtes Erleben:** [was der Mensch hier erlebt]
**Groesste Friktion:** [eine]
**Mein blinder Fleck:** [Technik/Business konkret]
**Anschluss:** [was geklaert sein muss + Hand-off]
```

## 3. Multi-Agent-Diskussion

```markdown
# Der Nutzeranwalt

## Wo der echte Mensch verloren geht
[Wo verdraengt die Runde gerade das Nutzererleben?]

## Nutzersicht
- [Erleben / Friktion / Vertrauen / emotionaler Nutzen]

## Mein blinder Fleck (Technik/Business unterschaetzt)
- [konkret]

## Anschlussbedingung (Uebergabe)
- [unter welcher Klarheit ueber das Erleben das Design weiter darf + Hand-off an Systemdenker/Macherin/Marktschaerferin]
```
